## Emergent insights & cross-feature threads

### The one design law, re-discovered under a dozen names: structural impossibility over guard-piles
Across every cluster the project keeps converging on the same move — redesign so the unsafe outcome *cannot be expressed*, rather than detecting it after the fact. `extra='forbid'` makes a bad migration LOUD instead of silently dropping keys (001); gate on actual-rendered-size not stamped intent (012); pick the dirty-signal the success path *clears* (`node.program is None`), not the derived value a reset keeps coherent (029); content-addressed edits make silent mislocation impossible by construction (036→039). The canonical counter-example is the five 038 brace-structure guards, all deleted by 039. **So-what:** lift one high-altitude heuristic to `conventions.md ## Design decisions` — *"if you're adding a second wave of guards to second-guess what an actor MEANT, the contract is unsound; redesign so the safe outcome is structural, then delete the guards"* — and make it the first question in any spec review whose decision section is mostly validation logic. It would have short-circuited the 036→038→039 detour (two wasted guard waves) at 038.

### Two opposite rules about speculative machinery — the corpus only resolves the contradiction across clusters
"Cut speculative API surface at design review" (AppContext deleted 002, `can_export`/`preview_bytes` dropped, `ctx.state` deleted 041) coexists with "keep latent-but-unused machinery dormant if cheap and reversible" (010 FitPolicy enums kept). Derived from foundations 001-011 + copilot design 020.03 + script 040-045. The reconciling axis is stated exactly once (010): the test is **"is REMOVING it churn?"** — surface you must *teach or maintain* (an ABC method, a model-visible enum, a curated namespace whose every omission is a NameError) gets cut; inert latent code with zero teaching/maintenance cost and a named near-future consumer stays dormant. **So-what:** file the disambiguated rule, or the maintainer re-litigates it every feature (002, 010, 041 each re-derived it from scratch).

### "Mirror the existing sibling" — most productive AND most dangerous default
It is the source of every reusable seam (commands.py → ShaderLibFileManager → CopilotCapabilities → ProjectSession; CopilotBridge → GateChannel as "the bridge inverted") **and** a recurring bug class, because the copy silently drops the sibling's asymmetry. GateChannel copied CopilotBridge but forgot `reopen()` → every confirm instant-cancelled (020.17). `connect_youtube` mirrored Telegram's two-step flow into a dead second tool (020.21). The export-isolation per-caller bracket mirrored a pattern that re-introduced the exact miss-a-caller risk a funnel existed to prevent (041). **So-what:** add a mirror-checklist to the feature flow — when reusing a sibling pattern, enumerate the sibling's asymmetries (re-arm/reopen, lifecycle hooks, step count, every caller of the funnel) and prove each is carried or provably unneeded. Skipping it costs a class-1 bug found only in live verification.

### Every LLM-facing lesson is a corollary of one actor-model
The model has exactly two reliable behaviors: **it copies text verbatim**, and **it is blind to anything outside its token stream.** Content over coordinates (036/039); token-stream match but don't *advertise* invariance (020.13); feed back the *sanitized* text it actually produced (020.20); facts-as-data succeed but facts-as-conscience fail (035); tool-count-must-not-grow because a lazy model won't call an optional inspect tool (033); the better-model babysitter test to reject overfit guards (029/037). These are not six lessons — they are six corollaries of one model of the actor. **So-what:** there is no single home for this; it's scattered across ~12 specs and partly in the dogfood skill. **This is the single highest-leverage knowledge-lift in the corpus** — one `copilot/LLM-agent-design` skill that states the actor model once and hangs every corollary off it.

### The premise never checked against the runtime produces the most expensive cascades
They cluster at two moments: drafting infrastructure from a *remembered* model of the engine, and inheriting a behavior claim from training data or a sibling. The 020.20 array-uniform guard rested on a false "silent off-thread pop" premise a one-line moderngl probe disproved — and that single wrong premise was the HEAD domino of a multi-failure agent cascade (blocked correct `set_uniform` → illegal GLSL edit → compile error → `replace_lines` thrash to cap → confabulation). The 027 long-running server was designed by 3 of 4 agents on a false "state is expensive to rebuild" premise a code-read killed. **So-what:** the global "verify the premise" rule is filed, but the highest-yield instantiation is specific — a single library *probe* or code-read in the first 10 minutes of **spec drafting** (not impl) invalidates whole designs. Require a code-grounded devil's-advocate agent specifically when a *majority* of design agents agree (027 proves consensus is not evidence: one grounded contrarian overturned three).

### Token-cost was redesigned more than any feature except the edit tool — a measurement-literacy gap closed the hard way
Billed input is the SUM across iterations (the cost driver); context-fullness is iteration-0 input (the compact driver); the two were repeatedly conflated. 025's gauge shipped reading the summed rollup (would peg full on any tool turn — caught at review); 028 measured 25k avg input with tails to 231k from verbatim history; the analyzer dropped error-turn cost (40% under-report); cost verdicts reversed cold→warm on a 67% cache hit. The whole memory model swung from too-little (023 confabulation) to too-much (028, same source 10x). **So-what:** file three measurement facts as one "agent cost measurement" known-quirk (billed=sum; fullness=iteration-0; cache-hit dominates re-run cost so never gate on a cold sample). A visualization must answer exactly ONE named question (028's "when do I compact?") — if you can't name the question, you're showing data not signal.

## Recurring mistakes
Ranked by frequency × cost.

### 1. Spec'd safety/guarantee ships as a no-op (scaffolded-but-unwired)
**Pattern:** a safety mechanism is specified, named, sometimes given config/comments/flags in code, but the wire that makes it fire is never connected. It reads as protected but isn't, and is caught only at post-impl review or live use — never by the normal test pass.
**Instances:** 001 (Decision-14 file-lifecycle promises all unimplemented); 020.12 `max_edit_retries` dead config read nowhere (a failing edit retried to `max_iterations`, burning money); 020.15 `copilot_turn_active` editor-lock flag + "# locks read-only" comments, read by nothing; 022 the per-completed-turn save (the *headline durability promise*) never wired — only switch/shutdown/clear saved; 033 oscillation brake + lib-edit coverage shipped with ZERO tests, checklist premise false.
**Root cause:** reviewers check intent against the spec; nobody verifies the *consumer* side (who READS the flag / who CALLS the save). A named field + a comment + a config key substitutes for proof that it fires.
**Cheapest guardrail:** a spec-completion checklist item — for every safety flag/guarantee, name the exact call site that READS it and the test that fails if the wire is cut. "Defined" ≠ "wired"; the verification step must exercise the consumer, not the producer. **Mechanically greppable** (grep the symbol for a reader), which is why it's the strongest dev_flow-checklist candidate.

### 2. Stale/intended state trusted instead of ground-truth state
**Pattern:** correctness is gated on a remembered/stamped/intended value (closure capture, persisted-but-unvalidated credential, UI slider, stale snapshot, mtime, insertion order) that silently diverges from live reality.
**Instances:** 009 stale-looks-connected (migrated `user_id` persisted without re-validation); 009 stale/revoked token showing green "Connected" over a raw API error; 010 silent-truncation lie (sliders `prepare()` overrode); 012 stale-preset closure (corrected to actual-vs-expected size); 029 cached-vs-live source dirty check that NEVER fires (`CompileUnit.empty` keeps source coherent; real signal is `node.program is None`); 030 disk-is-not-live-state (blind dir-copy captures stale `node.json`); 044 precedence relying on dict insertion order perturbable by the `(path,mtime)` reorder.
**Root cause:** the stamped value is on-hand at the gate; the live value requires re-derivation. First-pass design reasons from "what I set" not "what is true now," and the divergence window is invisible until live use.
**Cheapest guardrail:** any gate must key on a value whose lifecycle the SUCCESS path owns (`program is None`, content-hash, actual artifact size), never a derived/stamped/order-incidental value; the falsifier test must accumulate divergence on the LIVE object then assert.

### 3. Guessing from training-data recall instead of verifying the installed dependency
**Pattern:** an API shape or runtime behavior is taken from memory/upstream-docs/a sibling idiom and is wrong against the pinned package or real runtime — a crash, a silently-broken feature, or a whole guard built on a false premise.
**Instances:** 004 imgui 1.91-vs-1.92 misses (`text_colored` arg-swap, WindowFlags enum, the 1.92 font rework — upstream-fetch not install+grep); 018/019 `imgui.Mod_ctrl` doesn't exist / `set_window_focus(name)` segfaults / `invisible_button` not a nav stop; 020.23 array-uniform guard on a false "silent off-thread pop" premise a moderngl probe disproved (head domino of a multi-failure cascade); 026 D2 single-thread monkeypatch wrong vs prod threading; 031 pyright won't satisfy a named-param Protocol with a Callable-field fake; 041 empty `__builtins__` assumed safe but `class` needs `__build_class__` from builtins.
**Root cause:** recall and a working sibling are cheaper than reading installed source or running a probe, and they "sound right." Pinned-version-vs-upstream and test-idiom-vs-prod gaps are invisible until a crash.
**Cheapest guardrail:** already filed (verify against `.venv/site-packages`); strengthen as a pre-impl gate phrased "install+grep or runtime-probe, re-probe at review, NOT upstream-fetch/recall/sibling-idiom" for every dependency API and claimed runtime behavior a guard or design rests on.

### 4. Speculative abstraction built before the 2nd real consumer, then torn out
**Pattern:** an abstraction designed from N=1 or hypothetical needs, built, then deleted/collapsed once a real second consumer (or none) shows it misfits.
**Instances:** 002 AppContext 15-field bundle built then deleted; 001/003 `can_export`/`AuthState 5→3`/`preview_bytes` cut; 009 credential-migration lift specced then deleted entirely; 020.02-03 `node_ops.py` floated pre-consumer, flagged speculative; 041 `ctx.state` channel + two-phase tick reserved then deleted; 045 curated math exec-globals maintained across 040/041/044 then dropped for plain Python; 020.14 line-editing tools fully built with ~136 tests then retired by content-addressing.
**Root cause:** the in-house conventions make "design the seam up front" feel mandatory, so the team over-anticipates the second consumer's shape, combined with a strong instinct to generalize from the first example.
**Cheapest guardrail:** the N=2-on-paper test enforced at design review (write the second real consumer's full field list; <30% lands inside = wrong axis); for solo-tool features prefer "make the user redo setup once" over a migration; keep latent machinery only if removing it later is provably zero churn.

### 5. Guard-pile retrofit onto an unsound contract
**Pattern:** a contract forcing the actor to reproduce something it's bad at (coordinates, a stamped intent) is patched with successive waves of intent-guessing guards; each wave leaves "the next ring out" until the contract is redesigned and the guards deleted.
**Instances:** 020.14→036→038→039 source-edit addressing (all guards + line/anchor machinery deleted for content-addressing); 034/038 anchor machinery + the empty-`new_text` whole-function-delete false-reject; 024 comment-loss guard that blocked intentional rewrites (inspected only one side of a normalized match); the `in_flight`/busy-flag guards (009 stuck `in_flight`; 022 `_cancel` overloaded for teardown AND user-Stop).
**Root cause:** a targeted guard makes the observed bad trace stop, so it feels like progress; the contract's unsoundness is only visible once the guard count keeps climbing and false-rejects appear.
**Cheapest guardrail:** a stop rule — when adding a SECOND wave of guards to suppress silent-corruption/false-reject corners of the same contract, that is the signal to redesign the construction (loud-by-construction), not add guard #6.

### 6. Prototyped/specced against the lucky case
**Pattern:** a path is reasoned through on a happy input and crashes on a realistic edge: empty collection, non-ASCII char, tuple-vs-list shape, a config map born inconsistent, comment-only/array-typed value.
**Instances:** 012 channel-less account `IndexError` on `items[0]`; 008 tuple-vs-list dedup never matched + the tuple-vs-list text crash `706f1c5` + scalar int to a float-asserting branch; 013 NUMBER-branch lexer crash on unicode digits (violated its own never-crash contract); 031 `SUPPORTED_MEDIA_EXTENSIONS` (2) vs `IMAGE/VIDEO_EXTENSIONS` (7) born inconsistent → `.webm` KeyError swallowed → node vanishes; 034 comment-only `old_str` lexes to zero tokens; 041 integer uniforms never reached the GPU (coerce gl_type-blind, moderngl raised, swallowed at DEBUG).
**Root cause:** first-pass reasoning runs the modal/easy input in the author's head; the degenerate class isn't enumerated, and a swallowing except/skip-log hides it from tests.
**Cheapest guardrail:** at value/type boundaries enumerate the degenerate set explicitly (empty, unicode, tuple-vs-list, array-typed, comment-only, the second suffix map) and add one falsifier per class; **ban silent swallow** — a coercion/lookup failure must surface a typed error, not a DEBUG skip-log.

### 7. Cross-cutting invariant enforced per-call-site, so a caller is missed
**Pattern:** an invariant that must hold on EVERY path of a fan-out is wired at individual call sites; a sibling is forgotten, and the same root bug gets fixed twice before moving to the shared funnel.
**Instances:** 041 export-isolation per-caller bracket left GUI Render/Share un-wrapped (post-impl BLOCKER), fixed by the `Node.render_media` funnel; 028 F04 pointer clobber fixed in smoke.py then re-clobbered by the pytest fixture (real fix: one provenance gate in `App.__init__`); 016 `.trash/` filter on one glob but not the watcher's → infinite rebuild loop; 018 `_check_invariants` popup sum missed `is_shader_lib_picker_open`; 029/031 parallel name-keyed side-dicts (a repo-wide 10-instance sweep).
**Root cause:** per-call-site enforcement is the path of least resistance when adding the guarantee to the first caller you touch; the other callers are out of view, with no single test asserting coverage.
**Cheapest guardrail:** put the bracket at the single shared funnel (or lift the fact onto the entity) + ONE invariant test asserting coverage across all paths/entities. A second fix of the same bug at a sibling site is the trigger.

### 8. Locked-decision-vs-shipped-reality drift (append-rot)
**Pattern:** a spec locks a shape/mechanism that is reversed once rendered or re-grounded, but the locked section is left stale describing the abandoned approach.
**Instances:** 013 locked decision 8 still described the reverted hover-callback; 016 D1/D2/D12 locked a Manage-files section rejected mid-impl as "slop"; 012 abandoned `draw_target_panel` layout; 020.11 §3 tool catalog superseded before shipping (claims `list_nodes` GL-free; a docs corpus that didn't exist); 029 spec's "`_TOOL_VERBS` drifts" motivation contradicted by git (born complete).
**Root cause:** specs are append-only living docs; on a mid-flight reversal the energy goes into the new code, and reconciling the now-false locked text is deferred and forgotten.
**Cheapest guardrail:** make "reconcile reversed locked sections to as-built (correct in place or banner SUPERSEDED) in the same wave" a sanitize/cold-context checklist item.

### 9. Trace-driven overfit guard against a transient model/transport quirk
**Pattern:** a single live failing transcript motivates a guard/prompt-rule that fixes one instance of model carelessness or a provider quirk rather than a class derivable from the pipeline's design.
**Instances:** 029 overfit tribunal cut 5 of 6 trace-motivated guards; 037/035 "better-model test" logs a model-competence flaw as a DATUM not a prompt rule; 033 reasoning-suppression assumed `effort:minimal` honored (empirically false → raise budget instead); 035 in-flight D9/9x root-cause attribution WRONG, corrected by a post-run forensic swarm.
**Root cause:** a live failing trace is vivid and tempts patching every observed instance; without a generality bar, model carelessness and transport quirks get encoded as permanent guards.
**Cheapest guardrail:** gate every reactive guard on "would a strictly BETTER model still need this? is it a CLASS from our pipeline's design, not an INSTANCE of carelessness?"; never change a destructive op's behavior on a heuristic guess; adopt the stopping rule.

### 10. Review anchored to the wrong artifact (stale proto / not-yet-written code)
**Pattern:** a pre-impl reviewer returns FAIL on a false premise — it read an outdated prototype, the un-written engine code, or test pseudo-code as if it were the spec under review.
**Instances:** 040 round 1 REJECTED (false positive, read the proto as the spec); 045 Reviewer 2 FAIL on the un-written engine code; 044 reviewer FAIL on "test pseudo-code not in the spec"; 041 re-swarm caught a MIS-FRAMED "make check fails" headline (a ruff version-pin skew, not a gate failure).
**Root cause:** review prompts don't pin the exact artifact-under-review and its boundaries, so a reviewer grabs the nearest concrete code.
**Cheapest guardrail:** already covered by the `review-agent-loop` skill (late-round false-positive detection, anchor-to-named-artifact); reinforce by stating the exact spec file + "this code does not exist yet / ignore the stale proto at path X" boundary in every reviewer prompt.
