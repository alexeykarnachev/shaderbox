## Goal

Make a uniform a first-class object you attach a behavior SCRIPT to: a small CPU-side script computes the uniform's value every frame, turning ShaderBox from a static-uniform playground into the first step of a mini game-engine. v1 ships the engine core only ‚Äî a new `shaderbox/scripting/` package that ProjectSession ticks once per frame to write computed values into each `Node.uniform_values` BEFORE `Node.render()` reads them. The script ON DISK is body-only Python: the user writes just the function body and hands the result back through a passed-in `out` object (`out.set(...)`), with `ctx` (the read-only world) also in scope. Binding is by filename (`nodes/<id>/scripts/u_<name>.py` ‚Üí uniform `u_<name>`). The language is a swappable backend behind a `Behavior` protocol so a future C/native backend drops in with no engine-loop change. Errors are data, never a crash: a broken script freezes the uniform at its last-good value and records a `ScriptError`. A prototype at `/tmp/sb_engine_proto/` validated the shape (body-only `out.set()`, curated namespace, error-as-data with the SyntaxError pointing at the user's source line, scoped live‚Üîexport determinism, ~3-4us/tick).

## Design decisions (numbered; lock-in only)

> Convention form: "we decided X; revisit if Y." The five plan-lock open questions were resolved by the maintainer ‚Äî see the *Plan-lock resolution log* at the end.

1. **`EngineContext` shape + the two-phase contract.** A single frozen-ish dataclass `EngineContext(t: float, dt: float, frame: int, mouse: tuple[float, float] = (0.0, 0.0), state: dict[str, Any] = {}, uniforms: dict[str, Any] = {})` is the one object every script receives. `t/dt/frame` are the clock (live: from `glfw.get_time`; export: deterministic `t = i/fps`, `dt = 1/fps` ‚Äî see decision 4). `mouse/state/uniforms` are RESERVED for 041/042 ‚Äî present in the shape so the protocol is stable, but v1 leaves `mouse=(0,0)` and `state={}`. The tick runs **two phases**: A (state evolution ‚Äî mutates `ctx.state`; v1 STUB, no state objects) then B (uniform compute ‚Äî writes `node.uniform_values[name]` for each scripted uniform). The invariant: **A runs to completion before any B reads `ctx.state`, and the whole tick runs before `Node.render()` reads `uniform_values`.** v1 runs only B. **Determinism is SCOPED, not universal:** a script that is a pure function of `ctx.t` (e.g. `sin(ctx.t)`) yields the same value at the same `t` in live and export. A script that reads `ctx.dt` or the prev-frame value via `ctx.uniforms` (an integrator/smoother ‚Äî the proto's `u_follow`) is PATH-DEPENDENT and WILL diverge between live (variable dt) and export (fixed `1/fps`). This is acknowledged, not a bug: the determinism guarantee holds only for `ctx.t`-pure scripts; integrators are frame-rate-dependent by nature. The verification (unit + dogfood) asserts determinism ONLY on a `ctx.t`-pure script, and a separate test documents the integrator-divergence as expected. Revisit the context fields if 042's pong needs state shapes a `dict[str, Any]` can't carry.

2. **Body-only on disk; value handed back via an OUTPUT OBJECT, NOT a last-expression rewrite (resolves blocker 1 ‚Äî locked Q2).** The script file is the function BODY only ‚Äî no signature/imports persisted, ONE representation on disk, no dual-representation sync (the 039 lesson). The engine executes the body with two names in scope: `ctx` (the read-only world ‚Äî `EngineContext`) and `out` (the value sink for THIS uniform). The user assigns the result by calling `out.set(...)`:
   ```python
   # scripts/u_wave.py
   out.set(0.5 + 0.3 * sin(ctx.t))
   # scripts/u_offset.py  (vec2)
   r = 0.4
   out.set(r * cos(ctx.t), r * sin(ctx.t))
   # scripts/u_color.py  (branching is natural ‚Äî no wrapper edge cases)
   if ctx.t > 1.0:
       out.set(1.0, 0.0, 0.0)
   else:
       out.set(0.0, 0.0, 1.0)
   ```
   **The proto at `/tmp/sb_engine_proto/` STILL uses the OLD "rewrite the last line to `__out = (‚Ä¶)`" approach (engine.py:122) ‚Äî it is OUTDATED relative to this spec and must be updated to the `out.set()` shape before it's cited as validation (a Files-touched/verification item below).** That rewrite is REJECTED here: it was a hidden source transform that broke on a statement/comment/multiline-expr last line and pointed a `SyntaxError` into the WRAPPER, not the user's source (the 039 ghost). The output-object design removes that class BY CONSTRUCTION: the engine NEVER rewrites the body ‚Äî it `exec()`s it verbatim with `ctx` + `out` injected into the namespace dict (injection is a namespace entry, NOT a source edit), so a `SyntaxError`/runtime error's `lineno` already points at the user's source ‚Äî no AST surgery, no line-remap needed.
   **How `out` knows the uniform shape (the metadata gap):** `out` is a `UniformOut` object the ENGINE constructs per scripted-uniform per-tick, injected with the live `moderngl.Uniform` (so it holds `(dim, array_length, gl_type)`). `out.set(*args)` accepts a scalar (`out.set(0.5)`), per-component vec args (`out.set(x, y, z)`) or a single sequence (`out.set((x, y, z))`); it normalizes `*args` to a single value then validates the shape against its uniform via the reused coercion (decision 7 ‚Äî `coerce_uniform_value(value, self._uniform)`), and on mismatch records a `ScriptError` (kind=`runtime`) with the `shape_hint` message, freezing last-good. The body never sees the `moderngl.Uniform` ‚Äî only `out.set`. **Not calling `out.set()` is valid** ‚Äî the uniform simply holds its previous value that frame (a no-op tick; useful for "update only when there's something to say"); a dedicated test covers this. **Multiple `out.set()` calls** in one tick = last-wins (documented, not an error). Revisit only if a uniform genuinely needs MULTIPLE outputs from one script (then `out` grows named sinks ‚Äî not a reshape of the single-uniform contract).

3. **`Behavior` backend abstraction (language-swappable).** A `Behavior` Protocol with `compute(ctx: EngineContext, out: UniformOut) -> None` is the seam that makes the script LANGUAGE a detail (the script writes its result into `out`, decision 2). v1 ships `PythonBehavior` (compile the raw body once into a code object ‚Äî no AST rewrite, decision 2 ‚Äî and `exec()` it each tick against a curated namespace with `ctx`/`out` injected). A future C/native backend (`CBehavior` loading a `.so`, calling its entry that fills `out`) implements the SAME protocol ‚Äî the engine loop iterates `Behavior` objects and never knows the language. A compiled `Behavior` caches its own compile-phase `ScriptError` (a `SyntaxError` in the body makes a `Behavior` that always freezes). Revisit if a backend needs a richer contract (e.g. a phase-A `update_state(ctx)` ‚Äî that's 042, a SECOND protocol method, not a reshape of `compute`).

4. **Export must route through the engine tick (resolves blocker 2).** Today `core.py::_render_video` and `_render_image` call `Node.render()` DIRECTLY, bypassing the engine ‚Äî so an exported video would have FROZEN scripted uniforms while live preview animates (silent live‚Üîexport divergence). Export MUST tick `(t, dt, frame)` before each `node.render()`. We do NOT pass ProjectSession into Node (breaks the 025 boundary). The locked seam:
   The boundary constraint: `Node` (App-free, 025) must not import or hold `ProjectSession`; the engine reaches the export frame loop via a callback the session injects onto `Node` at load. (Two shapes were weighed ‚Äî a ProjectSession wrapper that owns the export loop vs. a callback on `Node` ‚Äî and the callback won because `_render_video`/`_render_image` already live on `Node`, so the loop stays there and only the per-frame tick is injected.)
   - **LOCKED (Q1)** ‚Äî a tick‚Üírender seam the engine owns, via the injected callback. `_render_video`/`_render_image` (core.py:401/489) call `Node.render()` directly TODAY; they must instead tick-then-render per frame. **Caller graph (verified ‚Äî this is the real blast):** `render_media`‚Üí`_render_media_into`‚Üí`_render_video`/`_render_image` is reached from THREE places: the Render tab (`tabs/render.py:60` ‚Üí `node.render_media`), the share scratch path (`tabs/share_state.py:99` `render_to` ‚Üí `render_media`), and the COPILOT render tools (`copilot/backend.py:909/935/1000` ‚Üí `share_state.render_to`). So the seam must sit where the per-frame `Node.render(u_time=...)` call is made ‚Äî INSIDE the export loop (`_render_video` 489-499, `_render_image` 401-423) ‚Äî and the engine tick runs there.
   - **The hook fires ONLY in the export loop ‚Äî NEVER in `Node.render()` (resolves the round-2 double-tick risk).** `Node` gains an optional `on_pre_render: Callable[[float, float, int], None] | None` the session sets at load; it is invoked by the EXPORT entry points (`_render_video`'s per-frame loop before each `self.render(i/fps, ...)`; `_render_image` before its single `self.render(u_time=..., ...)` ‚Äî passing that frame's `t/dt/frame`), and is **NOT called from `Node.render()` itself.** This is critical: the LIVE path already ticks via `session.tick()` in `ui.py::update_and_draw` (before the render block); if `render()` ALSO fired the hook, the live frame would tick TWICE (once in ui.py, once in render) ‚Äî a uniform-value corruption. So: **live ticks once (ui.py `session.tick`); export ticks once per frame (the export-loop hook); `Node.render()` never ticks.** The copilot render reaches `_render_*` via `share_state.render_to`‚Üí`render_media` and is bridge-marshalled to the main thread (`backend.py run_on_main`), so its tick is a main-thread CPU write ‚Äî no threading hazard.
   - **Tick scope = every node that will render THIS frame, matching the live render-gate (resolves the round-2 scope question).** `ui.py::update_and_draw` renders the current node always, and all nodes when `is_render_all_nodes` (ui.py:135-141). `session.tick()` must tick exactly that set (the same gate), so a scripted uniform on a non-current node animates identically live and in export. Export ticks the single node it renders. (v1 keeps this simple: tick the same nodes the live frame renders; don't tick popup-suppressed frames, matching the existing render skip.)
   - Rated **HIGH blast-radius** (a multi-caller hot-path touch), not MED ‚Äî reflected in Review sizing + the blast table.

5. **`set_uniform` rejects a script-driven uniform (resolves blocker 3).** A script writing `u_x` every tick would silently no-op a UI slider and the copilot `set_uniform` (`backend.py::set_uniform`). The copilot `set_uniform` MUST reject a script-driven uniform with a clear, model-facing error ‚Äî mirroring the existing engine-driven rejection at `backend.py` (`if name in ENGINE_DRIVEN_UNIFORMS: ‚Ä¶ "is engine-owned ‚Ä¶ cannot be set"`) and the sampler/block rejection just below it. The check needs ProjectSession to expose "which uniforms on node N are script-driven" (the engine owns that set per node); `set_uniform` queries it via an injected getter (same pattern as the existing `get_ui_nodes` etc. getters on `CopilotBackend`). The UI slider visual indicator is DEFERRED to 041 (Out of scope). Revisit the message/seam if 043's write-behavior tool needs the inverse (attach a script TO a uniform that currently has a hand-set value).

6. **Binding by FILENAME; no `node.json` entry.** `nodes/<id>/scripts/u_<name>.py` binds to uniform `u_<name>` ‚Äî the filename IS the binding, stateless. `UINode.save` writes NO scripts entry into `node.json` and `UIUniform` gains NO persisted binding field (confirmed: `ui_models.py::UINode.save` serializes uniform VALUES + `ui_state`; a script binding is not uniform state, it's a sibling file). The engine resolves script‚Üíuniform at load by globbing `scripts/u_*.py` against the node's active uniforms. Revisit if a binding ever needs metadata the filename can't carry (then it's a manifest, and that's a deliberate reversal).

7. **Scriptable uniform kinds + reuse of copilot coercion (home LOCKED ‚Äî Q4).** v1 scripts target the uniform kinds a Python value can produce: scalar (`float`/`uint`, `dim==1, array_length==1`), small vector (`vecN`, `dim in 2..4, array_length==1`), and the existing array kinds the coercion already handles (`float[N]`, the `uint[N]` text array). Samplers and UBOs are NOT scriptable. `out.set(...)` (decision 2) validates + shapes its arg by REUSING `_coerce_uniform_value` (do NOT reinvent coercion): a value that doesn't fit the uniform's `(dim, array_length, gl_type)` is a runtime `ScriptError` carrying `_set_uniform_shape_hint`'s message, frozen at last-good. **LOCKED: `_coerce_uniform_value` + `_is_number` + `_coerce_array` + `_set_uniform_shape_hint` move to a NEW leaf `shaderbox/uniform_coerce.py`** (public names, drop the leading underscore) that both `copilot/backend.py` and `scripting/` import ‚Äî a dedicated leaf reads cleaner than growing `util.py`, and breaks the cycle (the engine must not import `copilot/`). Revisit the scriptable-kind set when a real sampler script need appears.

8. **`ScriptError` as data + last-good freeze (mirrors the shader-error precedent).** A frozen `ScriptError(uniform_name: str, kind: Literal["compile","runtime"], message: str, line: int = -1)` mirrors `shader_errors.ShaderError` (error-as-data; `line` is the user-source line via the decision-2 remap, `-1` when unmappable). A compile failure (bad AST / `SyntaxError`) makes the `Behavior` permanently error until the file changes; a runtime failure (`exec` raises, coercion returns None) is caught per-tick. EITHER way: the uniform FREEZES at its last-good value (its current `node.uniform_values[name]`, or the node-intrinsic default if never computed ‚Äî via `Node.seed_uniform_values`), the frame CONTINUES, and the error is recorded on the engine state (`dict[uniform_key, ScriptError]`) for 041's UI to surface. Never raises into the frame loop. Revisit if a third error kind earns a branch (e.g. a "shape-mismatch" distinct from generic runtime ‚Äî only if 041's UI wants to render it differently).

9. **Hot-reload cache keyed by `(path, mtime)`; watcher lives IN the engine (Q5 LOCKED).** The compiled `Behavior` is cached keyed by `(path, mtime)`; on a poll, recompile ONLY scripts whose mtime changed ‚Äî never recompile all scripts every frame. A new/removed `u_*.py` file is also detected (glob + stat, like `maybe_rebuild_lib_index`). **LOCKED: the poll method lives on the script engine (`scripting/engine.py`), invoked from `ui.py::update_and_draw`** (after the lib/node watch, before render) for live, and run once at load for export ‚Äî keeping all scripting machinery self-contained rather than scattering a script-specific watcher into `watch.py`. Revisit if per-frame glob+stat over the scripts dir shows up in a profile (then cache the dir listing + invalidate on a coarser signal).

10. **ProjectSession placement (App-free, 025 boundary).** The engine is REPO code owned by the headless `ProjectSession` (`project_session.py`) ‚Äî it imports no imgui/glfw and constructs no window/context (the 025 invariant). ProjectSession gains: the per-node script registry (compiled `Behavior` objects + their `(path, mtime)` cache), the per-node `ScriptError` map, the (v1-empty) `state` dict, and the `tick(t, dt, frame)` method that runs phase A (stub) then phase B (writes `node.uniform_values`). Scripts are PROJECT DATA (live under the project's `nodes/<id>/scripts/`); the engine is repo code. App's frame loop calls `session.tick(...)` before render (live); export routes through the session per decision 4. Revisit if the engine grows state that genuinely needs to live closer to GL than the headless core (it should not ‚Äî the engine is pure CPU).

11. **New `shaderbox/scripting/` package shape.** A new leaf-ish package (imports `shader_errors` for the error precedent + the shared coercion home; imports NO imgui/glfw/App, NO `core.Node` type beyond what it needs to write `uniform_values` ‚Äî it works against the `node.uniform_values` dict + `get_active_uniforms()`, not the GL program). Proposed modules:
    - `scripting/context.py` ‚Äî `EngineContext` dataclass (leaf).
    - `scripting/errors.py` ‚Äî `ScriptError` (leaf, mirrors `shader_errors`). (May merge into `engine.py` if it stays tiny.)
    - `scripting/behavior.py` ‚Äî the `Behavior` Protocol + `PythonBehavior` (the namespace, the `compile()`/`exec()` seam, the `UniformOut` sink + its shape validation via the reused coercion).
    - `scripting/engine.py` ‚Äî the script registry per node, the `(path, mtime)` cache, the load/resolve-against-uniforms + orphan/mismatch warning, and the `tick` body (phases A stub / B compute) that `ProjectSession.tick` drives. `ProjectSession.tick` is the public entry; the package internals are not reached by UI/App directly.
    - The curated namespace (`math`-only, no numpy ‚Äî decision/Out-of-scope) lives in `behavior.py` as a module-level builder.

12. **The exec() seam needs ONE sanctioned `# type: ignore` (or `# noqa`).** Raw `compile()` + `exec()` with a curated namespace, NO sandboxing (personal IDE, projects are not shared, max perf ‚Äî the locked posture). `exec` against a dict trips ruff's `S102` and possibly a pyright `Any`-flow note; this is the ONE sanctioned suppression for the engine seam. It MUST be added to the `conventions.md ## Known quirks` sanctioned-suppression allowlist in the same wave (with the rationale: "the scripting engine's `exec()` of a user body ‚Äî no-sandbox is a locked decision, not a stub"). No OTHER suppressions land. Revisit only if the no-sandbox posture is ever reversed (it won't be for a personal IDE).
